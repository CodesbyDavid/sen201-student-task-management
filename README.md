## 1. Introduction
The Student Task Management System is a simple software application designed to help students manage their academic tasks. It allows users to add, view, update, and delete tasks.

---

## 2. Software Development Life Cycle (SDLC)

### 2.1 Requirement Analysis
The system requirements include:
- Adding new tasks
- Viewing all tasks
- Updating task status
- Deleting tasks

Non-functional requirements include:
- Ease of use
- Fast response time
- Reliability

---

### 2.2 System Design
The system is designed using:
- **Programming Language:** Python
- **Interface:** Command-line
- **Data Storage:** In-memory list

---

### 2.3 Implementation
The system is implemented using Python functions to manage tasks.

---

### 2.4 Testing
Testing was carried out by:
- Adding tasks
- Viewing tasks
- Ensuring correct output

---

### 2.5 Deployment
The application can be deployed on any computer system with Python installed.

---

### 2.6 Maintenance
Maintenance includes fixing bugs and adding new features such as task deadlines.

---

## 3. Conclusion
The Student Task Management System demonstrates the proper application of all phases of the Software Development Life Cycle.
